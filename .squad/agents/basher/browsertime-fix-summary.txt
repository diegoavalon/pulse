BROWSERTIME COLLECTION GRACEFUL FAILURE FIX
============================================

PROBLEM:
--------
GitHub Actions run #26734701410 failed with:
  "Error: No browsertime JSON found for page 'homepage' (desktop/mobile)"

Root cause: ehealthinsurance.com blocking Docker container requests (403 errors)
            → sitespeed.io runs but produces no browsertime.json
            → collect-sitespeed.mjs throws hard error
            → workflow fails, NO artifacts preserved (including screenshots)

SOLUTION:
---------
Implemented graceful degradation across the artifact collection pipeline:

1. collect-sitespeed.mjs (lines 57-76)
   - Changed hard throw to console.warn() for missing browsertime.json
   - Made HAR and Coach JSON collection conditional with warnings
   - Allows workflow to continue and upload partial artifacts

2. consolidate-sitespeed.mjs (lines 66-81)
   - Updated findRunDirs() to accept directories with screenshots OR browsertime.json
   - Previously only found directories WITH browsertime.json
   - Now discovers partial runs for processing

3. Extractor (no changes needed)
   - Already handles missing browsertime.json gracefully
   - Returns error result that consolidation logs as warning
